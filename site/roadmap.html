GIS Capture & Integration Roadmap

Indoors → Outdoor Infrastructure → Underground Utilities — SF Rec & Park Infrastructure Assessment Program

Phased plan: Indoors/Facilities → Outdoor Infrastructure → Underground Utilities.

Phase 0 — Foundation (runs before Phase 1, maintained throughout)

These decisions must be locked in before bulk capture begins in any phase — retrofitting them later is far more expensive than settling them once, up front.

  • Control network / RTK service: select one RTK correction provider and standardize on it for every capture stream.
  • Coordinate system standard: California State Plane (WKID 7132) horizontally, NAVD88 vertically — applied consistently across indoor, outdoor, and utility data.
  • ID / naming governance: hierarchical Site → Facility → Level → Unit IDs for buildings, extended with sibling Hardscape and Utility ID conventions under the same Site ID.
  • Enterprise geodatabase schema design: Indoors dataset (Sites/Facilities/Levels/Units/Details), plus new Hardscape and Utility feature datasets, all in one versioned enterprise geodatabase.
  • FME workspace templates: reusable E57→LAS georeferencing workspace, extendable to future CAD/BIM imports.
  • Pilot selection: choose one representative building + adjacent hardscape as the pilot to validate the full pipeline before scaling portfolio-wide.

PHASE 1 Indoors — Buildings & Facilities

Objective

Build a validated ArcGIS Indoors dataset for every Rec & Park building, with floor-aware maps and oriented imagery ready for the Indoors Viewer and your GIS hub.

Capture

  • Primary: iPhone LiDAR + PIX4Dcatch/PIX4Dmatic (default for routine buildings).
  • Secondary: Polycam for quick single-room documentation; Realsee kept in rotation where already convenient.
  • Rent a dedicated scanner (BLK2GO/NavVis) only for unusually large/complex facilities.

Processing & Integration

  • FME: E57/point cloud → georeferenced LAS (control-point transform tied to the RTK network).
  • ArcGIS Pro: Generate Floor Plan From Point Cloud → manual cleanup → Import Features to Indoor Dataset.
  • Oriented imagery: Import Indoor Images for 360° panoramas per floor.
  • Attribute enrichment: room USE_TYPE, webTMA asset cross-reference IDs, occupancy/space-planning fields.
▶ Gate to Phase 2: pilot building fully validated end-to-end (capture → import → published map) before scaling to the full building portfolio.

PHASE 2 Outdoors — Hardscape & Site Infrastructure

Objective

Document courts, trails, plazas, parking, and site amenities as standard outdoor GIS asset layers (NOT part of the Indoors interior data model), tied to the same Site IDs established in Phase 1.

Capture

  • Gap to close: aerial imagery capability for efficient large-area orthomosaic capture — met via existing EagleView/Spexi service accounts rather than an in-house drone.
  • RTK-corrected GNSS + Survey123 for discrete point features (benches, light poles, drain inlets, playground equipment, signage).
  • Polycam / phone 360 for quick condition-assessment documentation of individual amenities.
  • PIX4Dcatch on large flat hardscape areas (courts, plazas, large paved paths): per PIX4D’s own capture guidance, walk straight lines with gentle turns, keep parallel walking paths spaced roughly 2.5 meters apart, hold the device at a fixed slight tilt rather than pointing straight down — pointing straight down can hurt satellite signal acquisition when an external RTK device is connected. See the Field Capture Methodology page for the full official Do’s/Don’ts reference.

Schema & Integration

  • New Hardscape feature dataset (separate from Indoors schema) — polygons for paved areas/courts, lines for trails, points for amenities — all keyed to Site ID.
  • Condition-scoring attributes built into the schema from day one, cross-referenced to webTMA asset IDs.
▶ Gate to Phase 3: hardscape schema and pilot site published and validated; imagery workflow proven repeatable by staff.

PHASE 3 Underground — Utilities: Water & Sewer

Objective

Document Rec & Park-owned utility infrastructure and integrate reference access to city-owned utility networks, without duplicating data SFPUC already maintains.

Step 0 — Resolve ownership (before any capture spend)

  • Determine which utility infrastructure is Rec & Park-owned (irrigation, small distribution lines, on-site sewer laterals) versus SFPUC-owned (water/sewer mains).
  • For SFPUC-owned infrastructure: pursue a data-sharing agreement to bring existing utility GIS layers in as read-only reference layers — do not recapture.

Capture (Rec & Park-owned assets only)

  • GPR (ground-penetrating radar) or professional utility locating for undocumented/buried lines — contracted service, not in-house.
  • RTK GNSS + Survey123 for visible surface features: valves, hydrants, cleanouts, manholes, meter boxes.
  • Coordinate with 811/USA utility locates before any invasive fieldwork.

Schema decision

Default to a simple asset inventory schema (point/line feature classes with attributes). Only adopt the full ArcGIS Utility Network model if there's a specific tracing/isolation-analysis requirement.

▶ Program complete when Indoors, Hardscape, and Utility layers all share consistent Site IDs, CRS, and metadata standards under one enterprise geodatabase.
