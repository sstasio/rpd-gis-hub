ArcGIS Enterprise Integration

How the MCP server authenticates against and queries an on‑premises (or VPC‑hosted) ArcGIS Enterprise portal + server.

Integration flow

Auth: generateToken

Enterprise portals typically use the classic token endpoint with a named service account, or SAML/PKI in federated environments.

POST https://gis.yourcity.gov/portal/sharing/rest/generateToken
Content-Type: application/x-www-form-urlencoded

username=SERVICE_ACCOUNT
&password=********
&referer=https://gis.yourcity.gov
&f=json

1. Find a layer (portal search)

GET https://gis.yourcity.gov/portal/sharing/rest/search
  ?q=title:"Facility Assets"
  &f=json
  &token={portal_token}

2. Query features

GET https://gis.yourcity.gov/server/rest/services/Parks/FacilityAssets/MapServer/0/query
  ?where=1=1
  &outFields=*
  &f=geojson
  &token={portal_token}

Operational considerations

Network reachability

The MCP server must reach the Enterprise portal + server, typically via VPN, private endpoint, or reverse proxy.

Certificates

Internal CAs are common — the Node/Python HTTP clients may need a custom CA bundle in production.

Token lifetime

Enterprise tokens are short‑lived by default; the server refreshes proactively before expiry.
